# 07. Security, Privacy, and License Policy

## 1. 개인정보 원칙

- PDF 파일은 기본적으로 외부 서버로 전송하지 않는다.
- 광고, 추적, 분석 SDK를 포함하지 않는다.
- crash report도 기본 비활성화한다.
- 로그에는 문서 내용, OCR 텍스트, 암호, 토큰을 기록하지 않는다.
- 최근 파일 기록은 사용자가 끌 수 있어야 한다.
- 최근 파일 전체 경로 마스킹 옵션을 제공한다.

## 2. 파일 보안

- PDF는 신뢰할 수 없는 입력으로 취급한다.
- 파일 크기 제한과 timeout을 둔다.
- 외부 명령 실행은 allowlist된 바이너리만 허용한다.
- shell 문자열 실행 금지. 반드시 argument array 사용.
- 임시 파일은 앱 전용 temp 폴더에 생성한다.
- 저장 전 출력 파일을 검증한다.
- overwrite 전 백업 옵션을 제공한다.

## 3. 암호/보안 기능

허용:

- 사용자가 암호를 입력해 암호화 PDF 열기
- 사용자가 알고 있는 암호로 암호 제거
- 새 암호 설정
- 문서 권한 설정

금지:

- 암호 brute force
- DRM 우회
- 권한 제한 우회
- 소유자 암호 우회
- 접근 제한 회피

## 4. 업데이트 보안

- 업데이트 파일은 서명 검증 후 설치한다.
- 업데이트 manifest는 HTTPS에서 가져온다.
- 개인 키는 로컬 개발자 PC 또는 GitHub Secrets에 보관하고 저장소에 커밋하지 않는다.
- checksums를 함께 제공한다.
- 업데이트 실패 시 기존 버전을 보존한다.
